# 免费图书分享小程序 需求文档

> 📅 **生成时间**：2026-05-01
> 🎯 **核心目标**：为用户提供一个简洁的免费电子书浏览、搜索、下载工具
> 🔧 **技术栈**：Taro 4 + React 18 + TypeScript + NutUI-React + SCSS

## 1. 项目概述

- **核心问题**：400+ 本免费电子书分散在百度网盘文件夹中，用户难以快速找到想要的书籍
- **目标用户**：有电子书阅读需求、习惯使用百度网盘下载的用户
- **产品价值**：将分散的书籍资源结构化展示，用户通过分类浏览或关键词搜索，3 步内获取下载链接
- **成功标准**：用户能在 10 秒内找到目标书籍并复制下载链接

## 2. 竞品与创新点

**主要竞品：**

| 竞品 | 优势 | 不足 |
|------|------|------|
| [微信读书](https://weread.qq.com/) | 海量正版书籍、社交阅读体验 | 在线阅读为主，不支持下载自有资源 |
| [BookChat](https://gitee.com/truthhun/BookChat) | 开源、支持分类浏览和在线阅读 | 面向程序员群体，通用性不足 |
| [共享图书小程序](https://developers.weixin.qq.com/community/develop/article/doc/000aec37aecad842767b969e756013) | 支持电子书下载和纸质书借阅 | 功能重、UI 老旧 |

**差异化方向：**

1. **极简获取链路** — 竞品多侧重在线阅读，本项目聚焦"找到 → 复制链接 → 下载"的最短路径，无广告、无注册、无多余功能
2. **精选策展** — 不是海量搜索结果，而是经过筛选的 400+ 本高质量书籍，按学术/文学/历史等 16 个分类精准组织

## 3. 功能需求

### 分类浏览（P0）

**用户场景**：用户打开小程序，想看看"历史"类有哪些书，点击分类查看列表，找到感兴趣的点击查看详情

**验收标准：**
- 首页展示 16 个分类卡片，显示分类名和书籍数量
- 点击分类进入列表页，展示该分类下所有书籍
- 列表页支持上拉加载更多（每页 20 本）
- 书籍卡片显示封面缩略图、书名、作者、简介摘要

**业务规则：**
- 分类数据从七牛云 CDN 按需加载，已加载的分类缓存在内存中
- 图片使用七牛云缩略图参数 `?imageView2/2/w/200/h/280/q/85`

---

### 搜索（P0）

**用户场景**：用户知道想找的书名或作者，通过搜索快速定位

**验收标准：**
- 首页顶部搜索栏，输入关键词后 300ms 防抖触发搜索
- 搜索范围为所有分类的书名和作者（不区分大小写）
- 搜索结果以列表形式展示，复用书籍卡片样式
- 无结果时显示空状态提示

**业务规则：**
- 首次搜索时批量加载所有分类数据（并发请求），后续走内存缓存
- 最小搜索长度为 2 个字符

---

### 复制下载链接（P0）

**用户场景**：用户在书籍详情页点击"复制链接"，粘贴到百度网盘获取资源

**验收标准：**
- 详情页展示"复制下载链接"主按钮
- 点击后复制到剪贴板，Toast 提示"已复制，请打开百度网盘粘贴下载"
- 复制内容格式为 `链接：https://pan.baidu.com/s/xxx\n提取码：xxx`

**业务规则：**
- 从 `bd_link` 中解析 `pwd=` 参数获取提取码
- 无提取码时仅复制链接

---

### 收藏功能（P1）

**用户场景**：用户浏览到感兴趣的书但暂不下载，点击收藏方便后续查找

**验收标准：**
- 书籍卡片和详情页提供收藏按钮（心形图标）
- 首页底部或独立 tab 展示收藏列表
- 收藏数据存储在本地（`Taro.setStorageSync`）

**业务规则：**
- 收藏上限 100 本，超出时提示
- 无需登录，纯本地存储

---

### 分享给好友（P1）

**用户场景**：用户觉得某本书不错，想推荐给微信好友或群聊

**验收标准：**
- 详情页提供"分享给好友"按钮
- 点击后调用 `Taro.showShareMenu` / `onShareAppMessage` 生成分享卡片
- 好友点击分享卡片直接进入该书详情页

**业务规则：**
- 分享卡片展示书名、作者、封面缩略图
- 通过页面参数传递书籍 ID

---

### 浏览历史（P1）

**用户场景**：用户之前看过某本书但没收藏，想找回

**验收标准：**
- 自动记录用户进入详情页的书籍（最多 50 条）
- 首页提供"最近浏览"入口
- 支持清除历史记录

**业务规则：**
- 同一本书重复访问只保留最新一次记录
- 数据存储在本地（`Taro.setStorageSync`）

---

### 书单推荐（P2）

**用户场景**：用户不知道看什么，想看精选推荐

**验收标准：**
- 首页展示编辑精选书单（如"必读经典 Top10""职场进阶"）
- 点击书单查看书单内书籍列表

**业务规则：**
- 书单数据为静态配置，内置在小程序中
- 后续可扩展为远程配置

## 5. 技术方案

### 5.1 技术栈

- 前端：Taro 4.x + React 18 + TypeScript + SCSS
- UI 组件库：NutUI-React 3.x（按需引入）
- 状态管理：无（useState + useEffect + Context）
- 数据源：七牛云 CDN（静态 JSON 文件）
- 编译目标：weapp（先）→ h5（后）

### 5.2 数据模型

**Book（书籍）**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string (uuid) | 唯一标识 |
| name | string | 书名 |
| author | string | 作者 |
| type | string | 分类（16 个值） |
| picUrl | string | 七牛云封面图 URL |
| bd_link | string | 百度网盘分享链接 |
| brief | string | 简介 |
| addedAt | string (ISO 8601) | 入库时间 |

**CategoryIndex（分类汇总）**
| 字段 | 类型 | 说明 |
|------|------|------|
| type | string | 分类名 |
| count | number | 该分类书籍数量 |

**本地存储模型**

| Key | 类型 | 说明 |
|------|------|------|
| favorites | string[] | 收藏的书籍 ID 列表 |
| history | string[] | 浏览历史的书籍 ID 列表（按时间倒序，最多 50 条） |

### 5.3 数据流

```
App 启动
  ├─ 并发请求 configs.json（分类 URL 入口）+ index.json（分类汇总）
  ├─ 渲染首页分类列表
  └─ 用户操作按需加载：
      ├─ 点击分类 → 请求该分类 JSON → 缓存
      ├─ 搜索 → 首次批量加载所有分类 → 缓存 → 本地过滤
      └─ 图片 → 七牛云缩略图参数按需加载
```

无后端 API，所有数据通过 CDN 静态文件获取。

### 5.4 项目结构

```
src/
├── constants/
│   └── cdn.ts                 ← CDN 路径、分类配色
├── services/
│   └── data.ts                ← 数据请求层（含内存缓存）
├── utils/
│   ├── clipboard.ts           ← 复制链接、解析提取码
│   └── search.ts              ← 防抖、过滤
├── hooks/
│   ├── useConfigs.ts          ← configs + index 加载
│   ├── useCategory.ts         ← 单分类数据加载（含缓存）
│   ├── useSearch.ts           ← 跨分类搜索
│   ├── useFavorites.ts        ← 收藏管理
│   └── useHistory.ts          ← 浏览历史
├── pages/
│   ├── index/                 ← 首页（分类 + 搜索 + 最近浏览）
│   ├── category/              ← 分类列表
│   ├── detail/                ← 书籍详情
│   ├── favorites/             ← 收藏列表
│   └── booklist/              ← 书单详情
└── components/
    ├── BookCard/              ← 书籍卡片
    ├── CategoryCard/          ← 分类卡片
    └── SearchBar/             ← 搜索栏
```

### 5.5 关键考虑

- **域名备案**：七牛云测试域名 `qn-cdn.windlliu.com` 不可用于小程序正式上线，需绑定已备案域名
- **包体积**：小程序主包控制在 2MB 以内，图片全走 CDN 不打包
- **审核合规**：使用"复制链接"方式而非直接下载，审核通过率更高
