# 优化下载链接复制流程 - 功能开发文档

> 生成时间：2026-05-02
> 基于项目：resource-shop（免费图书分享小程序）
> 优先级：P1

---

## 1. 开发上下文

**技术栈**：Taro 4 + React 18 + TypeScript + SCSS + NutUI
**关联模块**：
- 剪贴板工具（`src/utils/clipboard.ts`）— 解析百度网盘链接、格式化、复制
- 详情页（`src/pages/detail/index.tsx`）— 复制按钮、引导面板
- 详情页样式（`src/pages/detail/index.scss`）— 引导面板样式

**现状**：用户点击"复制下载链接"后，复制内容为 `链接：xxx\n提取码：xxx`（分离格式），引导面板展示两步操作（打开百度网盘 → 粘贴链接到搜索栏）。用户需手动输入提取码。

**目标**：将 bd_link 解析后拼接为带 `?pwd=` 参数的完整 URL 一键复制，引导面板简化为"链接已复制，打开百度网盘粘贴即可"。

---

## 2. 验收标准

- [ ] 点击"复制下载链接"后，剪贴板内容为带提取码的完整 URL（如 `https://pan.baidu.com/s/xxx?pwd=xxx`）
- [ ] 无提取码时，只复制纯 URL
- [ ] 引导面板简化为：标题"链接已复制" + 链接展示 + 一句话引导 + 关闭按钮
- [ ] 详情页其他功能（收藏、分享、浏览历史）不受影响
- [ ] 构建无报错

## 3. 开发方案

### 3.1 文件清单

## 4. 实施步骤

**步骤 1：改造 clipboard.ts**
- [ ] 新增 `buildFullUrl(bdLink)` 函数：解析 bd_link，拼接为 `https://pan.baidu.com/s/xxx?pwd=xxx`
- [ ] 改造 `copyDownloadLink(bdLink)`：复制完整 URL 而非分离格式文本
- [ ] 移除 `formatDownloadText()` 函数（不再需要）

**步骤 2：简化详情页引导面板**
- [ ] 移除 `guideText`、`guideLink`、`guidePwd` 状态，替换为单一 `guideUrl` 状态
- [ ] `handleCopy` 简化：复制后直接设置 `guideUrl` 并显示面板
- [ ] 引导面板 JSX 简化：标题 + 链接展示 + "打开百度网盘粘贴即可" + 关闭按钮

**步骤 3：清理样式**
- [ ] 移除 `.guide-panel__steps-title`、`.guide-panel__step` 等不再使用的样式

**步骤 4：构建验证**
- [ ] `npm run build:weapp` 构建通过
