## 1. 开发上下文

**技术栈**：Taro 4 + React 18 + TypeScript + SCSS + NutUI-React
**关联模块**：CDN 常量（`src/constants/cdn.ts`）、构建配置（`config/dev.ts`、`config/prod.ts`）
**现有 UI 规范**：不涉及 UI 改动

**问题背景**：七牛云临时域名（如 `teaax7941.hn-bkt.clouddn.com`）不支持 HTTPS。开发和测试环境需要使用 HTTP 请求，正式环境绑定已备案域名后使用 HTTPS。

**当前状态**：
- `src/constants/cdn.ts` 中 `CDN_BASE` 硬编码为单个 `https://` URL，无环境区分
- `config/dev.ts` 和 `config/prod.ts` 的 `defineConstants` 均为空
- 应用代码中无任何 `process.env` 或环境判断逻辑
- `CDN_BASE` 被 `CONFIGS_URL`、`INDEX_URL` 引用，`services/data.ts` 和 `detail/index.tsx` 通过这些常量发起请求

## 3. 开发方案

### 3.1 核心思路

通过 Taro 的 `defineConstants` 在构建时注入 `CDN_PROTOCOL` 常量，`cdn.ts` 读取该常量拼接完整 URL。无需运行时判断，编译后直接替换为字符串字面量。

### 3.2 文件清单

- 修改：`config/dev.ts` — 添加 `defineConstants`，注入 `CDN_PROTOCOL = '"http"'`
- 修改：`config/prod.ts` — 添加 `defineConstants`，注入 `CDN_PROTOCOL = '"https"'`
- 修改：`src/constants/cdn.ts` — `CDN_BASE` 改为使用 `CDN_PROTOCOL` 拼接

### 3.3 新增依赖

无

## 4. 实施步骤

**步骤 1：修改构建配置**
- [ ] `config/dev.ts` 添加：
  ```ts
  defineConstants: {
    CDN_PROTOCOL: '"http"',
  },
  ```
- [ ] `config/prod.ts` 添加：
  ```ts
  defineConstants: {
    CDN_PROTOCOL: '"https"',
  },
  ```

**步骤 2：修改 CDN 常量**
- [ ] `src/constants/cdn.ts` 将：
  ```ts
  export const CDN_BASE = 'https://teaax7941.hn-bkt.clouddn.com/books-tidy/configs'
  ```
  改为：
  ```ts
  declare const CDN_PROTOCOL: string
  export const CDN_BASE = `${CDN_PROTOCOL}://teaax7941.hn-bkt.clouddn.com/books-tidy/configs`
  ```

**步骤 3：验证**
- [ ] 执行 `npm run dev:weapp`，在构建产物 `dist/common.js` 中确认 CDN_BASE 使用 `http://`
- [ ] 执行 `npm run build:weapp`，在构建产物中确认使用 `https://`
- [ ] 在微信开发者工具中测试首页分类加载正常

## 6. 约束与风险

- **构建时替换**：`defineConstants` 是编译期字符串替换，`CDN_PROTOCOL` 在产物中是字面量而非变量，无运行时开销
- **TypeScript 类型**：需用 `declare const` 声明全局常量，避免 TS 编译报错
- **正式上线**：需将 `config/prod.ts` 中的域名替换为已备案域名，同时协议保持 `https`
