## 场景描述
在网上发现了一批免费的电子图书，但是按书名来分布。数量有些大、且不方便按需选择。然后写了个工具进行整理，生成了下文格式的json数据
```json
{
    "id": "f9baf0fa-f207-45c5-95c8-3e66584ca1d8",
    "name": "世界经典学术集系列",
    "author": "列夫·马诺维奇",
    "type": "学术",
    "pic": "学术/世界经典学术集系列/9f368d20.jpg",
    "sourceFolder": "2000万人都学过：世界经典学术集系列（套装共13册） - 列夫•马诺维奇 等",
    "addedAt": "2026-04-30T09:28:58.158Z",
    "bd_link": "https://pan.baidu.com/s/1JI_fsCZfoC6qRiffWdguKQ?pwd=eg6n",
    "picUrl": "https://qn-cdn.windlliu.com/books-tidy/f9baf0fa-f207-45c5-95c8-3e66584ca1d8.jpg",
    "brief": "汇集全球经典学术著作，涵盖多学科前沿思想与理论成果。"
}
```
初始化大概有4、5百本的量, 想生成小程序、网站方便用户选择和下载。

## 想要的效果
技术倾向于 Taro + Ts => 生成小程序 和 h5网页
性能优化 => 按类别懒加载、图片下滑加载; 看下是否有成熟的插件 和其他方面优化
ui构建 => 尽量符合这个主题

主要功能是  
1. 用户按类型查看图书，
2. 每本图书展示 图片、作者、书名、简介等信息，点击按钮 获取 云盘链接
3. 有了云盘链接后(用户下载了云盘就直接打开云盘，没有下载就引导下载)
4. 某类型书籍过多，你判断下是分页还是说有更好的方式。
